# 管理后台界面边界

本文只定义“唯一统一管理后台”这一交付面的界面，不是整套产品的中心规划。创建软件、能力包选择、用户前台 UI、源码生成和扩展以 `product-scope.md`、`product-blueprint-and-generation.md` 与 ADR-0010 为准。

管理后台是工作型控制台，目标是快速查找、比较和处理数据，不做营销式页面。

## 信息架构

当前管理目录、路由和页面职责见 `admin-navigation.md`。本文件继续约束长期界面边界。

```text
顶部栏：当前软件下拉切换器 + 环境 + 当前官方/代理租户 + 管理员
左侧栏：当前软件已启用的公共能力目录
右侧区：当前目录对应的列表、表单和操作工作区

平台总览
├─ 全部软件指标
├─ 产品中心
└─ 系统状态

进入某款软件
├─ 产品总览
├─ 能力配置
├─ 用户与设备
├─ 代理/租户
├─ 权益与激活码
├─ 商品、订单与支付
├─ 版本与远程配置
├─ 文件与用量
└─ 管理员与审计
```

## 强制交互规则

- 顶部始终显示当前产品；跨产品总览使用明显的“全部产品”上下文。
- 软件少时可显示固定快捷入口，软件多时使用可搜索下拉选择器；禁止让软件名称无限横向挤压顶部栏。
- 左侧目录由产品能力配置生成；未启用的能力不显示，直接调用对应 API 也必须被服务端拒绝。
- 代理/租户是当前软件内部目录，不成为平台顶层中心；进入代理数据页时同时显示软件和租户上下文。
- 当前软件名称和上下文在所有写操作页面保持可见，防止管理员改错产品。
- 危险操作使用确认对话框并显示影响对象，不依赖颜色表达风险。
- 列表支持搜索、筛选、分页、空状态、错误状态和重试。
- 金额同时显示货币，时间明确时区，状态使用统一枚举文案。
- 密钥仅创建时显示一次，之后只显示前后缀。
- UI Page -> Feature Block / Hook -> API Client，禁止页面直接拼装底层请求。
- 所有管理员写操作显示结果并产生审计记录。
- “全部软件”和“具体软件”是两种明确上下文；切换时左侧目录整体替换，不把平台菜单与产品菜单混在一起。
- 第一阶段之外的能力不显示空菜单或“敬请期待”；能力达到当前阶段验收标准后再注册到目录。
- 创建软件使用固定五步工作流：基本资料、目标与能力、界面与配置、计划审阅、确认；服务端 ordinary 目录为空时显示真实空状态并禁用继续，不得注入 fixture 或空白能力包。
- 创建页面只能显示 API Client 返回的脱敏目录 ID/版本和 opaque 输出目标引用；不接受宿主路径、目录摘要或客户端 scope。experimental 使用独立受权路由且不出现在普通菜单。

## 前端目录方向

```text
admin-web/src/
  app/            路由、权限和全局产品上下文
  features/       可复用 Feature Block
  pages/          页面编排，不包含业务访问实现
  api/            由 OpenAPI 契约约束的 Client
  components/     无业务含义的基础组件
```
