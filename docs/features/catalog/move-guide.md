# Catalog 迁移与参考说明

旧项目中的会员表、价格字段和前端套餐卡只能用于盘点需求，不能直接复制为新 Catalog 数据模型。

## 概念映射

```text
旧会员套餐名称 -> Offer 的候选稳定身份
旧套餐时长/功能 -> 新建 OfferVersion
旧价格字段 -> 新建 PriceVersion，统一转换为最小货币单位整数
旧订单中的商品描述 -> 历史 CatalogSnapshot 或只读迁移快照
```

## 迁移步骤

1. 按 Product 和 Tenant 识别真实商品，清理重名、重复和已停用套餐。
2. 为稳定商品分配不可复用的 `offer_code`。
3. 把每组周期、权益和设备限制固化为 OfferVersion。
4. 把每个币种与生效区间固化为 PriceVersion，核对整数精度。
5. 为历史订单保存当时名称、金额和权益内容，不用当前 Offer 回填覆盖。
6. 在测试环境验证调价、下架和改名不会改变历史订单。

## 禁止做法

- 不把“会员等级”写入全局 User。
- 不允许前端直接传最终金额创建订单。
- 不修改已发布版本来修正历史数据。
- 不把成本价、销售价和代理结算价混成一个字段。
