# Order 迁移与参考说明

旧项目中“订单支付成功后直接修改会员”的单体流程不能直接迁移。

## 概念映射

```text
旧订单主表 -> 新 Order 购买事实
旧套餐名称/金额 -> 不可变 OrderItem 商品快照
旧支付单号 -> PaymentIntent / ProviderTransaction 关联
旧会员到期字段 -> Entitlement 历史来源，不写回 Order
```

## 迁移步骤

1. 分离订单状态、支付渠道状态和用户权益状态，不用一个 `status` 表达三者。
2. 核对历史订单币种、实付、退款和商品说明，无法确认的记录标记迁移风险。
3. 为历史订单建立只读 CatalogSnapshot；不使用当前套餐覆盖历史内容。
4. 用稳定外部引用连接 Payment 和 Entitlement，但不重放历史支付回调。
5. 对已支付但权益缺失、已退款但权益仍在等异常生成复核清单。
6. 验证重复导入不会创建第二笔订单或再次授予权益。

## 禁止做法

- 不信任旧客户端保存的支付成功标记。
- 不把订单完成直接等同权益已生效。
- 不覆盖原订单金额来表达退款。
- 不删除已取消、已过期或已退款订单的审计链路。
