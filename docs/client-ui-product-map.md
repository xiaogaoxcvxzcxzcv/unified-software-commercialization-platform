# 多端用户前台产品地图

本文档定义最终用户在桌面软件、Web、H5、微信小程序和手机 App 中看到的商业化公共前台。它不包含超管、运营、财务、代理管理员使用的管理后台。

## 1. 产品定位

用户前台是完整能力包面向最终用户的交付面。每款软件通过可信 `ProductContext + TenantContext`、能力包、UI Template 和主题 Token 组装所需入口。平台可以交付可维护的页面组合与接入源码，但不得为每款软件复制并分叉登录、会员、支付和计费等公共状态机。

UI Template 是一套可运行的用户前台呈现方案，包含 Shell、布局、导航、主题、公共页面编排和产品扩展槽，不只是皮肤。登录、注册、个人中心、会员权益等页面只有在对应完整能力包被选择、达到目标端 `available` 且模板声明兼容时才注册。平台不生成适用于所有软件的业务首页、业务目录页、工作台或核心内容；这些内容由软件开发人员在 custom 区域完成并接入。

```text
统一后端能力 -> 统一 SDK / API Client -> Client UI 契约
                                      ├─ Web / H5 组件包
                                      ├─ 桌面 WebView 组件包
                                      ├─ 微信小程序组件包
                                      └─ 手机 App 组件包

每款软件 = 自有核心业务 + 已启用的公共前台功能块 + 品牌主题 + 渠道适配
```

公共前台提供三种可按页面混合的交付形态：

- **版本化组件接入**：软件安装对应端的 Client UI 组件包，由蓝图生成默认编排，软件通过扩展点组合在自己的界面内。
- **Hosted UI 接入**：软件打开平台托管的登录、个人中心、套餐、结账和收银台页面，完成后通过一次性结果安全返回。
- **Generated Source**：生成器把页面组合、路由、入口、主题和渠道适配源码写入软件仓库，保留明确的 generated/custom 边界和版本锁。

三种形态共用同一契约和权威后端。例如桌面软件可以使用生成的个人中心组合源码，同时用系统浏览器打开托管登录和收银台。Hosted UI 的安全启动、回跳和渠道规则见 `platform/contracts/hosted-ui-contract.md`；源码所有权和升级见 `complete-capability-package-standard.md`。

## 2. 边界

### 用户前台负责

- 注册、登录、微信授权、退出和账号安全入口。
- 个人资料、当前会员权益、设备、订单和通知。
- 套餐比较、购买确认、收银台、支付结果和续费。
- AI 及其他按量能力的余额、用量、账单明细和 API Key。
- 产品帮助、客服、协议和隐私入口。

### 用户前台不负责

- 产品、租户、套餐、价格、模型路由和支付渠道的管理配置。
- 用户冻结、人工授予权益、退款审批、对账和审计查询。
- 代理经营、佣金结算、管理员角色和系统运维。
- 软件自己的创作、编辑、生产、项目管理等核心业务。

以上管理操作只进入管理后台；用户前台只展示当前用户可见的数据和允许发起的自助操作。

## 3. 公共信息架构

以下只描述公共能力页面的信息架构，不是每款软件的业务目录。不同端可以使用不同导航方式，但公共信息分组和业务语义保持一致；软件自己的业务目录由 custom 路由和导航扩展组成。

```text
公共用户前台
├─ 身份与进入
│  ├─ 登录 / 注册
│  ├─ 微信扫码或微信授权
│  ├─ 找回密码 / 二次验证
│  └─ 协议与隐私确认
├─ 个人中心
│  ├─ 账号资料与安全
│  ├─ 当前会员与权益
│  ├─ 我的设备
│  ├─ 我的订单
│  └─ 通知中心
├─ 会员与购买
│  ├─ 套餐卡片与能力对比
│  ├─ 购买确认
│  ├─ 收银台
│  └─ 支付结果 / 续费 / 升级
├─ 用量与开发者（按产品启用）
│  ├─ 额度与用量概览
│  ├─ 用量流水
│  ├─ API Key
│  └─ API 接入说明入口
└─ 支持
   ├─ 公告与消息
   ├─ 帮助中心
   └─ 客服入口
```

## 4. 黄金流程

### 4.1 首次进入与登录

```text
SDK 建立可信产品/租户上下文
-> 读取启用能力与登录方式
-> 密码登录 / 微信扫码 / 微信授权
-> 必要时完成协议确认或二次验证
-> 建立 UserContext
-> 检查当前产品权益
-> 有权益进入软件；无权益进入试用、兑换或购买入口
```

- 登录错误不能泄露账号是否存在。
- 登录完成后的返回位置由 `return_target` 控制，不能由外部 URL 任意跳转。
- 微信身份绑定、解绑和换绑必须有明确确认，不因一次扫码静默覆盖原账号。

### 4.2 选择会员并购买

```text
套餐卡片
-> 能力差异比较
-> 选择周期 / 规格
-> 服务端返回商品与价格快照
-> 用户确认金额、周期、权益和协议
-> 创建订单
-> 进入当前渠道收银台
-> 等待服务端确认支付
-> 展示支付结果
-> 查询最终权益
-> 返回原业务或个人中心
```

- 套餐卡只展示服务端返回的可售计划；客户端不计算最终价格。
- 支付成功动画不能代替权益检查；未获得最终结果时显示“确认中”，不可直接放行。
- 支持用户主动刷新支付状态、取消未支付订单和从订单详情继续支付。

### 4.3 个人中心自助管理

```text
个人中心概览
-> 查看当前权益、有效期、设备和额度
-> 进入资料 / 安全 / 设备 / 订单 / 通知
-> 执行允许的自助操作
-> SDK/API 返回结果
-> 更新当前摘要并保留可追踪结果
```

- 设备撤销需要显示设备名称、最后活跃时间及影响，并进行确认。
- 密钥只在创建成功时展示完整值一次，其后只展示摘要。
- 退出当前设备与退出全部设备必须是不同动作。

### 4.4 AI 用量与 API Key

```text
额度与用量概览
-> 按时间 / 模型 / 模态筛选
-> 查看 Token、图片、音频、视频、请求等统一计量维度
-> 进入单次请求明细
-> 必要时购买套餐或补充额度

开发者中心
-> 创建指定范围的 API Key
-> 完整密钥仅展示一次
-> 查看摘要、权限、创建时间和最后使用时间
-> 轮换或撤销
```

- 普通用户只看销售侧用量和费用，不显示 Provider 密钥、内部成本价、路由利润和其他租户数据。
- 新 AI 模态通过计量维度和逻辑模型元数据扩展，不重做整套页面。
- 用户端模型名称使用稳定逻辑名称；真实 Provider 或模型可由后台调整而不破坏前台。

## 5. 页面与能力启用关系

| 页面组 | 依赖能力 | 无能力时的行为 |
|---|---|---|
| 登录 / 注册 | identity | 产品无法进入账号态；按产品策略显示访客模式或不可用页 |
| 个人中心 | identity | 不注册入口 |
| 当前会员 | entitlement | 不显示会员入口；软件自有业务仍按产品策略工作 |
| 我的设备 | device | 不显示设备入口 |
| 激活码兑换 | license | 不显示兑换入口 |
| 会员套餐 | catalog + entitlement | 不显示购买会员入口 |
| 购买确认 / 订单 | order | 不允许发起购买 |
| 收银台 | payment + order | 显示不可购买状态，不展示无效付款码 |
| 用量与额度 | usage | 不显示用量入口 |
| API Key | developer-key + ai-gateway | 不显示开发者入口 |
| 通知中心 | notification | 不显示通知入口 |
| 用户文件 / 云同步 | storage | 不显示存储入口 |

前端隐藏入口不等于授权；服务端必须对关闭的能力继续拒绝访问。

## 6. 多端承载方式

| 端 | 建议承载 | 主要导航 | 关键适配 |
|---|---|---|---|
| Web | 响应式 Web 组件 | 顶部导航或账号侧栏 | 浏览器会话、微信扫码、网页支付回跳、深链接 |
| H5 | 移动优先 Web 组件 | 底部导航、全屏流程 | 微信内置浏览器、外部浏览器、窄屏收银台、安全区 |
| 桌面软件 | 内嵌 WebView 或独立账户窗口 | 软件入口 + 账户侧栏 | 系统浏览器登录回调、自定义协议、窗口关闭恢复、设备身份 |
| 微信小程序 | 原生小程序组件 | 底部 Tab 或页面栈 | 微信授权、手机号能力、小程序支付、订阅消息、页面栈返回 |
| 手机 App | Flutter / React Native / 原生组件 | 底部 Tab、页面栈 | App OAuth、原生支付、系统安全存储、深链接、应用商店规则 |

桌面和 Web 可以优先共享 React 业务组件；H5复用响应式版本。小程序和手机端不强求共享同一份渲染代码，但必须共用字段、状态、事件、设计 Token 和验收用例。

### 6.1 Hosted UI 适用方式

| 端 | 托管页建议 | 返回方式 |
|---|---|---|
| Web / H5 | 可直接使用托管登录、账户、套餐与收银台 | 已登记 HTTPS return target + 一次性 code |
| 桌面软件 | 优先用系统浏览器打开登录和收银台 | 自定义协议或 loopback + state + PKCE |
| 微信小程序 | 套餐说明、订单详情可托管；登录和支付必须原生适配 | 小程序页面栈/交互状态恢复，不依赖 Web Cookie |
| 手机 App | 登录可用系统认证会话；支付遵循原生渠道或商店规则 | Universal/App Link 或登记 Scheme + state + PKCE |

Hosted UI 只能使用服务端创建的短期交互地址，不能由软件通过 URL 自行指定产品、租户、价格和任意回跳地址。

## 7. 完全共用、渠道适配与软件自有

### 7.1 完全共用的契约与规则

- `ProductContext`、`TenantContext`、`UserContext` 和能力开关。
- 账号、权益、套餐、商品快照、订单、支付、设备、通知、用量和 API Key 数据契约。
- `idle / loading / ready / submitting / success / empty / failed / disabled` 状态。
- 金额、周期、权益结论、风险确认、错误码和完成事件语义。
- Logo、主色、强调色、圆角、产品名、帮助入口等主题 Token。
- 加载、空数据、失败、超时、重试、取消和成功后的业务流程。

### 7.2 必须按渠道适配

- 微信扫码、公众号、小程序和 App 的不同登录授权方式。
- Web 二维码支付、小程序支付、App 原生支付和桌面浏览器支付。
- Token / Cookie / 系统安全存储的会话保存方式。
- 外部浏览器回调、URL Scheme、Universal Link 和小程序页面栈。
- 推送、订阅消息、系统通知权限和应用商店购买限制。
- 文件选择、相机扫码、剪贴板和生物识别等设备能力。

渠道适配器只负责平台差异，不能决定价格、支付结果、权益或计费。

### 7.3 每款软件自己负责

- 软件核心业务首页、业务目录页、工作台、页面和工作流。
- 核心业务产生的内容、项目和业务数据结构。
- 在合适位置打开登录、购买、用量和个人中心的入口。
- 通过登记的 custom 路由、导航、槽位和完成事件把独有业务接入生成的前台框架。
- 产品 Logo、名称、品牌图片、主题 Token 和合规文案配置。
- 软件特有的权益功能映射，例如“批量导出”或“高清视频”。

每款软件不得自行复制并修改统一登录、订单、支付、权益、用量和 API Key 状态机。

### 7.4 源码所有权

- 公共 SDK、Headless 状态、业务 Feature Block 核心和统一后端由平台版本化维护。
- 蓝图可以生成页面组合、路由、入口、主题、渠道适配和扩展槽源码到软件仓库。
- 生成器只更新 lock 登记的 generated 区域，不覆盖 custom 独有业务。
- 需要修改公共实现时必须显式 eject，并接受停止自动覆盖、改用差异报告升级的结果。

## 8. 视觉与内容原则

- 参考现有后台的清爽白底、青绿色强调色、浅色状态块和紧凑圆角，但用户前台要比管理后台更聚焦任务。
- 会员卡可以突出推荐套餐，但不得用视觉手段隐藏价格、周期、自动续费或限制条件。
- 购买确认页必须同时展示产品、套餐、周期、实付金额、权益摘要和服务协议。
- 图标用于熟悉动作；危险操作需文字、影响说明和二次确认。
- 卡片圆角控制在 8px 以内，避免卡片套卡片；窄屏内容必须自然换行。
- 颜色不能单独承担成功、警告、失败和权益差异的表达。
- 所有文案由产品配置和本地化资源提供，业务字段不写死在组件中。

## 9. 组合方式示例

```text
工具型桌面软件
  登录 + 个人中心 + 会员套餐 + 收银台 + 设备 + 订单

AI Web 应用
  登录 + 个人中心 + 会员套餐 + 收银台 + 用量 + API Key + 通知

微信小程序
  微信授权 + 精简个人中心 + 套餐 + 小程序收银台 + 订单 + 通知

纯买断离线软件
  登录（可选）+ 激活码兑换 + 设备 + 永久权益 + 更新入口
```

## 10. 分阶段交付

### P0：账号与权益可用

- 登录、会话恢复、退出。
- 个人中心骨架、当前权益、设备、激活码兑换。
- Web / 桌面端第一套组件及契约测试。

### P1：商业闭环与 AI 用量

- 套餐卡、购买确认、收银台、支付结果和订单。
- 通知、用量概览、用量流水和 API Key。
- H5、小程序和手机端渠道适配。

### P2：规模化自助服务

- 用户文件、云同步、优惠券、发票申请等入口。
- 更完整的安全中心、消息偏好和多语言。

### P3：真实需求确认后

- 团队与席位、分销推广入口、海外支付和多币种。
- 代理白标用户站点及其独立域名体验。

## 11. 验收底线

- 同一组件在任何端都不能越过当前 `product_id + tenant_id + user_id` 范围。
- 禁用能力后入口消失，直接请求也被服务端拒绝。
- 支付、权益、设备撤销和 API Key 等关键流程覆盖加载、失败、超时、重试和最终一致状态。
- Web、桌面、H5、小程序和手机端至少各验证一次登录返回与支付返回路径。
- Hosted UI 必须验证 return target 白名单、`state/nonce`、一次性 code、PKCE、防重放和回跳丢失恢复。
- 旧产品主题或新增渠道不能改变已发布组件事件和字段语义。
- 管理员操作不得出现在最终用户前台中。
