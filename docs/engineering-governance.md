# 工程治理

## Definition of Done

一个功能完成必须同时满足：

1. 代码位于正确模块，依赖方向符合开发地图。
2. API、事件、错误码和安全说明已更新契约。
3. `docs/capability-index.md` 已更新。
4. 涉及可复用 UI 时更新 Feature Block Catalog。
5. 数据变更具有向前迁移和可执行的回退方案。
6. 单元、集成或契约测试覆盖正常与关键失败路径。
7. 核心流程通过冒烟测试，或记录未验证风险。
8. 幂等、超时、重试、审计和恢复策略已处理。
9. 日志不包含密码、令牌、支付密钥和个人敏感信息。
10. 文档为 UTF-8，旧参考目录未被修改。

## Contract First

以下工作必须先改契约再写实现：

- 前后端 API
- 模块间应用服务
- 领域事件
- 支付、短信、邮件、对象存储等 Provider
- 长任务、文件产物和客户端 SDK

契约至少说明：输入、输出、身份上下文、状态、错误、幂等键、事件、存储、重试、超时和安全要求。

## 数据规则

- 使用数据库迁移，已发布迁移不可改写。
- 金额使用最小货币单位的整数，不使用浮点数。
- 时间在服务端和数据库使用 UTC，界面按用户时区展示。
- 删除默认采用状态变更或软删除；法律或隐私删除另走清理流程。
- 关键业务状态使用数据库约束、外键和唯一索引保护。
- 订单、支付、权益流水采用不可变记录或保留完整变更历史。

## 可靠性规则

- 所有写接口定义幂等语义。
- 第三方回调必须验签、去重并可重复处理。
- 跨模块事件使用事务 Outbox，不能依赖“先写库再碰运气发消息”。
- 重试必须有最大次数、退避和死信处理。
- 每个外部调用设置超时；熔断和降级按风险引入。
- 每日备份不等于可恢复，必须定期执行恢复演练。

## 安全规则

- 密码使用成熟的自适应哈希算法，不自行实现加密。
- Access Token 短期有效，Refresh Token 可轮换、可撤销。
- 桌面客户端中的任何固定密钥都视为公开信息。
- 管理后台启用最小权限、敏感操作二次确认和完整审计。
- 支付结果只接受验签回调或服务端主动查询结果。
- 上传文件检查类型、大小、配额和访问权限，私有文件使用短时签名 URL。

## ADR

技术栈、模块边界、身份策略、支付 Provider、存储策略和核心模型发生长期变化时，必须在 `docs/adr/` 新增决策记录。旧 ADR 不删除，通过 superseded 状态保留历史。

## 测试分层

- Domain：纯业务规则单元测试。
- Application：事务、幂等和跨模块契约测试。
- Adapter：PostgreSQL、Redis、S3、支付沙箱集成测试。
- API：OpenAPI 契约与权限测试。
- UI：关键 Feature Block 组件测试。
- E2E：只覆盖少量黄金流程。

## AI 开发自检

每次变更前说明模块、涉及层、契约、ADR、测试和验证计划；变更后说明文件、索引、契约、测试结果、未验证项、UTF-8 与旧路径状态。

## 废弃策略

删除旧 API、字段或能力前，必须记录替代方案、数据迁移、兼容窗口、受影响客户端和计划移除时间。客户端 API 至少保留一个明确的迁移周期。

## 新软件接入变更边界

- 标准产品接入只新增产品配置、环境凭据和新软件侧 SDK 适配，统一后台共享代码默认只读。
- 接入任务需要修改共享 API、数据库或公共业务逻辑时，必须转换为独立的平台能力变更。
- Client API 同一主版本内只允许向后兼容新增，具体以 `platform/contracts/client-api-compatibility.md` 为准。
- 新产品上线前必须通过自身接入测试、产品/租户隔离测试和至少一款旧产品的关键流程回归。
