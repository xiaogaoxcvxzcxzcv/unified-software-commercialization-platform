# 软件独有功能扩展标准

统一底座负责重复能力，具体软件仍要拥有自己的核心业务。本标准确保两者可以共同出现，又不会让单款软件污染共享平台。

## Extension Manifest

每个扩展声明：稳定扩展 ID、所属 Product、版本、前台路由或槽位、后台目录项、权限、公开 API/事件、配置、迁移、测试和卸载/保留策略。

## 允许的扩展

- 在生成的前台组合中注册软件独有的业务首页、目录页、工作台、其他页面、导航和完成事件。
- 在统一后台为指定产品注册独有管理入口。
- 通过公开 SDK/API 调用通用能力，通过领域事件接收已授权事实。
- 在软件自有命名空间保存独有业务数据。

## 禁止的扩展

- 读取或写入平台其他模块的表或 Repository。
- 修改共享登录、支付、权益和用量状态机来照顾某个产品。
- 使用产品名称硬编码分支。
- 越过 Product/Tenant/Admin Scope 访问其他范围。
- 让重新生成或平台升级静默覆盖扩展源码。

## 提升为公共能力包

当第二款软件也需要同类扩展时，先检查能力索引和完整能力包目录。确认具有稳定共性后，另立平台能力变更，补齐九个交付面、兼容与回归；不能直接把第一款软件的代码复制到共享平台。
