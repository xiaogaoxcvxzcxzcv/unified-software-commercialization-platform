# ADR-0007：私有部署控制面与签名许可证

Status: accepted

Date: 2026-07-13

## Context

平台需要支持把某款软件或服务独立部署到客户环境并授权运行。代理租户表示统一云平台内的经营范围，普通激活码表示最终用户权益，两者都不能代表一套独立部署实例。

## Decision

- 新增 Deployment 模块，管理客户实例、环境、版本、实例指纹和运维状态。
- 新增独立的签名 Deployment License，使用非对称签名验证期限、功能、节点/席位/并发限制和离线宽限。
- 云端控制面只管理授权、升级元数据和允许的遥测；客户业务数据默认不回传。
- 控制面短期失联不能立即中断客户生产，许可证包含明确离线宽限。
- 私有部署许可证与用户激活码使用不同表、不同密钥和不同验证流程。

## Consequences

- 私有部署可以独立升级、备份、诊断和退役。
- 必须维护安装包、迁移包、兼容矩阵和离线授权流程。
- 第一位真实私有部署客户出现前，只实现契约和签名验证原型，不提前建设完整运维平台。

## Alternatives considered

- 把部署实例当 Tenant：无法表达独立数据库、版本和离线状态。
- 使用普通激活码：无法安全绑定实例和验证签名许可证。
