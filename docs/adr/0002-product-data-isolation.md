# ADR-0002：多产品数据隔离

- 状态：accepted
- 日期：2026-07-12

## 背景

同一个账号可能使用多款软件，但每款软件拥有独立套餐、权益、设备、订单、版本、配置、文件和额度。旧项目把套餐直接放在用户记录上，无法表达这一关系。

## 决策

- 用户身份全局共享，产品业务数据按 `product_id` 隔离。
- 权益使用 `user_id + product_id + grant` 建模，不在用户表保存全局会员状态。
- 客户端产品上下文由认证后的客户端身份确定，而不是信任请求参数。
- 数据库索引、Repository 方法、API 授权和对象存储路径都必须包含产品范围。
- 支持代理租户的业务数据在产品范围内继续按 `tenant_id` 隔离，详细决策见 ADR-0003。
- 自动测试必须证明产品 A 不能访问产品 B 的数据。

## 后果

- 用户可以一个账号登录全部软件，同时各产品权限保持独立。
- 所有新增业务表和查询都需要显式考虑产品范围。
- 跨产品统计必须走单独授权的管理查询。

## 备选方案

- 每个软件单独数据库和账号：否决，无法实现统一账号和统一运营。
- 在用户表增加多个套餐字段：否决，扩展性和一致性差。
