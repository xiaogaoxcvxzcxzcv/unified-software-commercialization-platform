# ADR-0003：租户从属于产品并承载代理功能

- 状态：accepted
- 日期：2026-07-13

## 背景

统一后台的中心是多款软件。每款软件需要官方经营和多个代理经营单元，代理之间的数据必须隔离。租户是代理能力的技术隔离单元，不应成为独立于产品的顶层中心。

## 决策

- Product 是一级业务主轴，Tenant 必须从属于一个 Product。
- 每个 Product 自动创建一个 `official` 租户，并可创建多个 `agent` 租户。
- 支持租户的权益、设备、激活码、订单、配置、文件和用量同时包含 `product_id + tenant_id`。
- ProductContext 由客户端身份确定，TenantContext 由官方渠道、代理分发证明、激活码或已认证绑定解析。
- 客户端任意提交的 `tenant_id` 不可信。
- 平台管理员可跨产品管理；产品管理员只管理当前产品；代理管理员只管理当前产品下自己的租户。
- 代理租户能力集中实现一次，由所有产品按能力配置启用。

## 后果

- 所有软件复用同一代理租户内核，不重复写后台代码。
- 官方直销和代理销售走同一业务链路，官方直销使用 official 租户。
- 基础隔离进入第一阶段；佣金、提现、白标和独立支付配置后置。

## 备选方案

- 把 Tenant 做成平台一级中心：否决，会偏离软件统一管理目标。
- 每款软件手写一套代理系统：否决，会重复代码并产生不同安全边界。
- 只在订单里保存代理 ID：否决，无法隔离配置、文件、用户和用量。
